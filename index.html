    Target Audience

    This API is intended for development resources well versed in mobile application development and integration.

    Introduction

    Welcome to the Gro Account Opening API! This document describes the means to integrate the Gro Account Opening Native Libraries into a mobile banking platform.

    Currently only iOS integration is supported, but development for Android and Windows is underway.

    GroAcct Class

    The GroAcct class library allows for initialization and rendering of the Gro Account opening solution. An instantiated GroAcct object will ultimately return a UIViewController subclass which should be added to the view hiearchy of the parent view container. The class library exposes properties and methods that allow the UIViewController and its root view to be configured and rendered propertly. Additionally data level properties are provided to pre-configure the Account opening workflow. Code examples are provided of the most common scenarios.

    GroAcct - UI Control

    The following properties define the intial rendering characteristis of the UIViewController and underlying root view.

    frame

    @property(nonatomic) CGRect frame

    Set the initial frame of the root view. This rectangle defines the size and position of the view in its superviewâ€™s coordinate system. You use this rectangle during layout operations to size and position the view. Setting this property changes the point specified by the center property and the size in the bounds rectangle accordingly. The coordinates of the frame rectangle are always specified in points.

    center

    @property(nonatomic) CGPoint center

    Set the intial center of the root view. The center is specified within the coordinate system of its superview and is measured in points. Setting this property changes the values of the frame properties accordingly.

    alpha

    @property(nonatomic) CGFloat alpha

    Set alpha of root view. The value of this property is a floating-point number in the range 0.0 to 1.0, where 0.0 represents totally transparent and 1.0 represents totally opaque. This value affects only the current view and does not affect any of its embedded subviews.

    headerTransparent

    @property(nonatomic) Boolean headerTransparent

    Default is false. If set to true the top banner of the pages in the Gro Account Opening workflow will appear transparent.

    footerVisible

    @property(nonatomic) Boolean footerTransparent

    Default is tue. If set to false the footer will not appear at the bottom of the screen.

    Personal Information

    The following string properties can be set to pre-populate data elements for single sign on situations, where the user has been authenticated through the mobile banking platform.

    Parameter Description
    ssn applicant’s ssn
    birthdate applicant’s DOB mm/dd/yyyy
    email Applicant’s email address
    phone Applicant’s mobile phone number (digits only, no dashes)
    firstName Applicant’s first name
    lastName Applicant’s last name
    middle applicants middle name or initial
    address1 Address 1 line
    address2 Address 2 line
    city city
    state state (2 character value)
    zip zip Code
    idNumber driver’s license or other ID number.

    Alternatively the setPersonalInformation method can be invoked which accepts a string of key value pairs seperated by an ampersand.

    setPersonalInformation('email=gro@example.com&firstName='John')

    Open Application

    The methods below initialize the Gro Account Opening application and return a UIViewController subclass.

    openApp()

    UIViewController openApp()

    This method with no parameters will initialize the Gro Account Opening application with all properties that have been set and return the coresponding UIViewController subclass.

    openApp(string ssoToken)

    UIViewController openApp(string SSOToken)

    This method will pass in a single-sign on token to initialize the application. The application will take the single sign on token, authenticate to an SSO server an pre-populate personal information of the applicant. The method will return an UIViewController subclass.

    Use Cases

    Below are the common use cases for opening the Gro Account Opening solution from a mobile banking application.

    Open App - No data initialization

    GroAcct myApp = new GroAcct();
return myApp.OpenApp();

    Open the Gro Account Opening Native App without any data values pre-populated.

    Open App - With products

    GroAcct myApp = new GroAcct();
myApp.productList = 'productID=24&productID=122'
return myApp.OpenApp();

    Open the Gro Account Opening Native App with specific products pre-selected in the shopping cart.

    Open App - SSO no Token

    
GroAcct myApp = new GroAcct();
myApp.ssn = '123454678'
myApp.firstName = 'John'
myApp.LastName = 'Smith'
mApp.email = 'smith@example.com'
myApp.OpenApp();

    Open the Gro Account opening app with applicant personal information pre-populated.

    If a user is already authenticated through the mobile application, the Gro Account Opening application can be invoked and passed application values that will appear pre-populated in the workflow pages.
