    GroAcct.Framework

    Welcome to the Gro Account Opening API Framework. This document describes the means of integrating the Gro Account Opening solution into a mobile banking platform. This API is in BETA and subject to change based on the outcome of early implementation efforts.

    Please note that there the mobile banking app may required configuration to interact with the GroAcct.Framework Currently only iOS integration is supported, but development for Android and Windows is underway.

    The GroAcct.Framework comes packages with a header file (GroAcct.h) which will expose one method for initializing and and rendering the application. During the execution of the account opening process, the framework will make calls to Gro web services. The mobile banking platform may need to be configured accordingly to allow such traffic.

    Target Audience

    This API is intended for development resources well versed in mobile application development and integration.

    showGroAcct

    The GroAcct.Framework exposes one method, showGroAcct. The method accepts a UIViewContoller object (or sub-class object) as input. If a UINavigationController object is passed to the showGroAcct method, a pushViewController method will be invoked, otherwise a presentModalViewController will be invoked to bring the account opening content to the foreground. The account opening content will be initialized with the provided set of configuration parameters.

    The showGroAcct method is an overloaded, static method with two options for supplying configuration parameters. One method is to pass in a string of keyValue pairs seperated by an ampersand. The second is to initialize an NSDictionary object of key-value pairs.

    (void) showGroAcct:(UIViewController *)viewController config:(NSString*)config

    (void) showGroAcct:(UIViewController *)viewController config:(NSDictionary*)config

    The showGroAcct method is void and returns no values. When the Gro Account Opening proces has terminated the application it will callBack to the mobile native banking application and return control to it.

    Configuration Parameters

    Below is the list of configuration parameters, all values are optional.

    Parameter Description
    ssoToken Single sign on token
    brandID Financial institution identifer
    memberID Membership number of the applicant
    ssn Applicant’s ssn
    birthdate Applicant’s DOB mm/dd/yyyy
    email Applicant’s email address
    phone Applicant’s mobile phone number (digits only, no dashes)
    firstName Applicant’s first name
    lastName Applicant’s last name
    middle applicants middle name or initial
    address1 Address 1 line
    address2 Address 2 line
    city City of Address
    state State of Address (2 character value)
    zip Zip Code of Address
    dlNumber driver’s license number.
    skipSMS true or false string value
    productID This value can appear multiple times and defines what products will be defaulted into the product show case
    skipIDCapture true or false string value

    Examples

    Open App - No data initialization

    #import <GroFramework/GroAcct.h>

NSString *config = nil;
[[GroAcct gro] showGroAcct:self config:config];

    Open the Gro Account Opening Native App without any data values pre-populated.

    Open App - With products

    
#import <GroFramework/GroAcct.h>

NSString *config = @"productID=24&productID=122";
[[GroAcct gro] showGroAcct:self config:config];

    Open the Gro Account Opening Native App with specific products pre-selected in the shopping cart.

    Open App - with personal Info

    #import <GroFramework/GroAcct.h>

NSDictionary *config = @{
    @"ssn" : @"111111111",
    @"email" : @"jon@example.com",
    @"firstName" : @"John",
    @"lastName" : @"Smith",
};
[[GroAcct gro] showGroAcct:self config:config];

    Open the Gro Account opening app with applicant personal information pre-populated.

    If a user is already authenticated through the mobile application, the Gro Account Opening application can be invoked and passed application values that will appear pre-populated in the workflow pages.
