    GroAcct.Framework

    Welcome to the Gro Account Opening API Framework. This document describes the means of integrating the Gro Account Opening solution into a mobile banking platform. This API is in BETA and subject to change based on the outcome of early implementation efforts. Currently only iOS integration is supported, but development for Android and Windows is underway.

    The GroAcct.Framework comes packages with a header file (GroAcct.h) which will expose one method for initializing and and rendering the application. During the execution of the account opening process, the framework will make calls to Gro web services. The mobile banking platform may need to be configured accordingly to allow such traffic.

    Target Audience

    This API is intended for development resources well versed in mobile application development and integration.

    ShowGroAcct

    The GroAcct.Framework exposes one method, ShowGroAcct. The method accepts a UIViewContoller object as input. The account opening content will be rendered inside of the view controller that is passed into the method and initialized with the provided set of configuration parameters.

    The ShowGroAcct method is an overloaded, static method with two options for supplying configuration parameters. One method is to pass in a string of keyValue pairs seperated by an ampersand. The second is to initialize a [[[[placeholder]]]] object of key-value pairs.

    void ShowGroAcct(UIViewContoller mwContainer, string config)

    void ShowGroAcct(UIViewContoller mwContainer, Map configMap) [[Placeholder, to be defined]]

    The ShowGroAcct method is void and returns no values. When the Gro Account Opening proces has terminated the application will close inside of the provided view controller returning control to the mobile bank framework through the viewDidDissapear event [[Need to discuss this]].

    Configuration Parameters

    Below is the list of configuration parameters, all values are optional.

    Parameter Description
    ssoToken Single sign on token
    memberID Membership number of the applicant
    ssn Applicant’s ssn
    birthdate Applicant’s DOB mm/dd/yyyy
    email Applicant’s email address
    phone Applicant’s mobile phone number (digits only, no dashes)
    firstName Applicant’s first name
    lastName Applicant’s last name
    middle applicants middle name or initial
    address1 Address 1 line
    address2 Address 2 line
    city City of Address
    state State of Address (2 character value)
    zip Zip Code of Address
    dlNumber driver’s license number.
    skipSMS true or false string value
    productID This value can appear multiple times and defines what products will be defaulted into the product show case
    skipIDCapture true or false string value

    Examples

    Open App - No data initialization

    include GroAcct.h

string config = null;
ShowGroAcct(myViewController, config);

    Open the Gro Account Opening Native App without any data values pre-populated.

    Open App - With products

    include GroAcct.h

string config = 'productID=24&productID=122'
ShowGroAcct(myViewController, config);

    Open the Gro Account Opening Native App with specific products pre-selected in the shopping cart.

    Open App - SSO no Token

    include GroAcct.h

string config = 'ssn='111111111&email=jon@example.com&firstName='John'&lastName='Smith'
ShowGroAcct(myViewController, config);

    Open the Gro Account opening app with applicant personal information pre-populated.

    If a user is already authenticated through the mobile application, the Gro Account Opening application can be invoked and passed application values that will appear pre-populated in the workflow pages.
